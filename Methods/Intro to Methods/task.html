Intro to Methods

Sometimes, you want to use the same block of code multiple times, at different places in your program. Methods, similar to the functions of other programming languages, can be used to insert a block of code wherever it's needed in your program. Here's what a method looks like:


  public class Main {
      // New method:
      static void exampleMethod() {
          System.out.println("I'm in a method!");
      }

      public static void main(String[] args) {
          // This is where you normally write your code
      }
  }

The main at the bottom and its code block is the same thing from all the theory lessons, where your code is run, but what do the other parts mean? static means that the method is part of the class itself and not an object of a class (we'll learn about classes and objects later). void means the method does not return anything (we'll learn about return types later in this lesson). exampleMethod is the name of the method, and can be more or less anything you want.

You may have noticed that the new method "exampleMethod" looks very similar to the "main" block of code you've been writing your code in during all of the theory lessons. That's because main is a method! So is solution in the programming tasks. Almost all of the code you write in Java is inside some sort of method.

So now we've defined exampleMethod, but by default it won't actually run, since it's not inside the main method where you normally write your code. To actually run the code in your method you have to call it. You call a method using the name you gave it:


  public class Main {
      // Define exampleMethod:
      static void exampleMethod() {
          System.out.println("I'm in a method!");
      }

      public static void main(String[] args) {
          // Only the code in this block is actually run by default

          // Call exampleMethod
          exampleMethod();
      }
  }

To call a method, just put parentheses after the method's name (and a semicolon to end the statement). When the program reaches this statement, it will jump to the method's block of code. Once that block has finished, the program will jump back to where it was before the call, and continue running. You can call the same method as many times as you want, from anywhere in your program. The above example will print "I'm in a method!" once and then end.

So, why do we use methods at all? If you want your program to print the same thing in different places, wouldn't it be easier to just copy and paste the print statement? Maybe, but if the method's block of code were much longer, it would make your program much longer than it needs to be. Even more importantly, if you copied and pasted the same bit of code many times, and then you need to change it (maybe you want to remove the exclamation point from "I'm in a method!"), you would have to go find every place you pasted the code and change it, which would be slow and error prone. If you use a method though, you only have to change it in one place.
